# Until - build & run

macOS menubar calendar app (Swift + SwiftUI/AppKit). Shows your next Google
Calendar event in the menubar, filtered by a user-authored JS expression.
The SwiftPM package, target, and product are **Until** (bundle id `app.until`).

## Install
```
swift build
```

## Build & run
```
swift run Until
```

For the full menubar experience (notifications, `LSUIElement`), run as a real
`.app` bundle instead — `dev.sh` kills the previous instance, rebuilds, and launches:
```
scripts/dev.sh
```

## Package a .app/.dmg
```
scripts/package-app.sh
open ".build/debug/Until.app"
```

## Distributable build (Developer ID + notarized)
```
scripts/release.sh
```
Builds release, Developer ID-signs with the hardened runtime, notarizes, and
staples. Needs a notarytool keychain profile (`until-notary` by default) — see
the header of `scripts/release.sh`. The app icon is generated by
`scripts/make-icon.swift` into `scripts/Until.icns` (delete it to regenerate).

## Google OAuth setup (one-time, by the developer)
1. Google Cloud Console → create a project → enable **Google Calendar API**.
2. Credentials → Create OAuth client ID → application type **Desktop app**.
3. `cp .env.example .env` and fill in `GOOGLE_OAUTH_CLIENT_ID` /
   `GOOGLE_OAUTH_CLIENT_SECRET` from the downloaded client JSON.
   - The build bakes these into the app's `Info.plist` (`package-app.sh` sources
     `.env`); `Models.swift` reads them at runtime, falling back to the process
     environment for the bare `swift run` path. `.env` is gitignored.
   - Desktop clients still need the secret at the token endpoint even with PKCE.
   - Uses the PKCE + loopback flow; any `127.0.0.1` port is allowed for desktop clients.
   - Scopes: `calendar.events`, `calendar.calendarlist.readonly`, `drive.file`,
     `userinfo.email`, `openid`. `drive.file` is the non-sensitive tier (avoids
     Google's CASA security assessment): the app only sees files/folders it
     created, which is enough here — meeting notes live in an app-managed
     "Meeting Notes" folder (found-or-created in My Drive), and templates are
     app-created Google Docs (copied per note). (`documents` is not needed:
     `drive.file` also authorizes the Docs API `batchUpdate` on app-created docs.)

## Layout
- `Package.swift` — SwiftPM package definition
- `Sources/Until/`
  - `AppDelegate.swift` — app bootstrap, menubar status item, popover, settings window
  - `Theme.swift` — design tokens (spacing/radius/surface) + shared UI components
  - `BrandIcon.swift` — the dot + chevron brand mark (menubar glyph + app icon geometry)
  - `Views.swift` — SwiftUI panel and preferences UI
  - `GoogleAuth.swift` — Google OAuth PKCE + loopback, token refresh
  - `CalendarClient.swift` — Calendar REST fetch + event normalization
  - `RuleEngine.swift` — native structured filter evaluator
  - `AppModel.swift` — config, polling, derived `AppState`, notification sync
  - `EventNotifier.swift` — native UserNotifications reminders
  - `ConfigStore.swift` / `KeychainStore.swift` — config JSON and Keychain token storage
  - `Models.swift` — shared app models and config

## Filter language
The Swift build evaluates structured rule conditions natively.
